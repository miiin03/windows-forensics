# 설계 명세 — 기말평가 루브릭 전 항목 5점 달성

날짜: 2026-06-15
대상 프로젝트: WinTrace (windows-forensics, 1팀 전체)
목표: 2026-1 네트워크보안 기말평가 루브릭 5개 항목 **전부 5점**

## Context (왜 이 작업을 하는가)

평가는 **1팀 전체 도구** 대상. 현 도구 = ActivitiesCache.db(Windows Timeline) 파싱 + 삭제 레코드 carving
(우리 `windows-timeline`) + Windows Event Log(EVTX) 분석 + ML 이상탐지 + 오프라인 LLM 조사관 + 통합 뷰어
(팀원 `wintrace`). 루브릭 대비 현재 추정 점수: #1=3, #2=4, #3=3, #4=2, #5=4. 가장 큰 갭 = #3(검증)·#4(발표).
작업 가능 기간 = 2주+ (종합·심화 가능). 코딩 범위 = 1팀 전체(우리 직접 + 팀원 조율).

## 도구 비교 기준 (무료/오픈소스 한정 — 사용자 결정)

대조는 **무료/오픈소스만**. 라이선스 장벽 0 → 실제 side-by-side 숫자 산출 가능.

| 구분 | 무료/오픈 도구 | 라이브 파싱 | 삭제 carving |
|---|---|---|---|
| ActivitiesCache 파서 | WxTCmd, DB Browser, WLEAPP, Autopsy | ✅ | ❌ |
| 범용 SQLite carver | FQLite, Undark, Bring2Lite, SQLite Deleted Record Parser | (raw) | ✅ (스키마 무지) |

**우리 차별점(무료 한정):** 무료 진영엔 "스키마 인지 ActivitiesCache carving" 부재.
범용 carver는 깨진 행을 raw로만 토함(AppId/Payload/타임스탬프 해석·activities 매핑·confidence 없음).
우리 = 4소스(freelist+freeblock+slack+WAL) carving + 31컬럼 Activity 의미 디코드 + EVTX·AI·통합·오프라인.

## 접근: 검증 우선 (Verification-first)

#3(검증)부터 한다. 거기서 나온 산출물(해시 무결성 / 무료도구 대조 / 오탐 정량화)이
#2 정확성·#1 의의·#4 발표를 동시에 끌어올림. "측정한 숫자 = 발표 근거".
순서: **#3 → #2 → #1 → #5 → #4(발표 마감)**.

## 항목별 설계 (5점 정의 → 갭 → 산출물)

### #1 문제 정의 & 포렌식적 의의 (3→5)
5점 = 대상·문제 구체 정의 + 실제 수사 시나리오·중요성 명확
- **D1.1** 1쪽 "문제·의의" 문서: 대상(ActivitiesCache+EVTX), 문제(삭제·은폐된 활동/보안흔적 복원),
  의의(무료진영 미지원인 스키마인지 carving을 통합 제공)
- **D1.2** 구체 수사 시나리오: 내부자/침해자가 활동기록+보안로그 은폐(history 삭제 + `wevtutil cl Security`)
  → freelist/WAL/slack 복원 + EVTX 1102 탐지로 은폐 역추적. named artifacts + 타임라인
- **D1.3** 무료도구 대비 차별점 표(슬라이드용)

### #2 기술 구현 & 정확성 (4→5)
5점 = 정확 동작 + 실데이터 실증 + 손상/엣지 견고
- **D2.1** OSError 버그 재현·수정: 통합 시 실DB 일부 레코드에서 발생(overflow/큰 payload 의심) + 회귀 테스트
  [우리 windows-timeline]
- **D2.2** 실데이터 실증 리포트: 실 ActivitiesCache.db + EVTX-ATTACK-SAMPLES → 복원수치·처리시간 문서화
- **D2.3** 손상/엣지 견고성: truncated/corrupt DB·malformed EVTX → graceful(크래시 없음) 테스트

### #3 포렌식 건전성 & 검증 (3→5) ★ 최대 레버리지
5점 = 무결성 보장 + 재현 + 상용도구 대조 + 한계·오탐 분석
- **D3.1** 해시·chain-of-custody: 증거파일(db/-wal/-shm/evtx) MD5+SHA256를 수집시점 기록
  → 출력 meta + `custody.json` 로그, 분석 후 재해시(무변경 증명). RO는 이미 됨 → 명시 assert+문서화
- **D3.2** 재현성 테스트: 동일입력→동일출력(결과 JSON 정규화 후 해시 대조)
- **D3.3** 무료 도구 대조(핵심):
  - WxTCmd ↔ 우리 라이브 파싱: 레코드수·필드 일치(기준선)
  - FQLite / Undark ↔ 우리 carving: 같은 DB 삭제복구 비교 → 복원수 동급+α & 스키마 디코드 우위
  - 대조 자동화 스크립트 + 결과표
- **D3.4** 오탐 정량화: 합성 ground-truth(insert N→delete M)로 carving precision/recall;
  브루트포스 휴리스틱 precision/recall(라벨 EVTX) [브루트포스는 wintrace 조율]
- **D3.5** 한계 문서: secure_delete/VACUUM 파괴, overflow 절단, WAL salt 세대, ML은 신호일 뿐, FP 조건

### #4 발표 & 전달력 (2→5) ★ 현재 최저
5점 = 논리 구성 + 명료 + 우수 데모 + 시간 준수 + Q&A 탁월
- **D4.1** 발표덱: 문제→의의→데모→기술→**검증(숫자)**→기여
- **D4.2** 동작 데모: Windows에서 exe 빌드(app.spec 준비됨) + 통합 뷰어(carved+EVTX+AI) 실연 + 백업 데모영상/GIF
- **D4.3** Q&A 시트: carving 신뢰성/오탐/무료도구 대비/무결성 예상질문+답
- **D4.4** 리허설(시간 준수)

### #5 창의성 & 기여도 (4→5)
5점 = 독창 + 까다로운 아티팩트/새 기법 + 실용 + 고른 팀 기여
- **D5.1** 새 기법 부각: 4소스 스키마인지 carving + 하이브리드 설명가능 AI(IsolationForest 신호 + LLM 해석, 오프라인);
  학술 근거(SQLite 복구 survey 2025, bring2lite DFRWS) 인용
- **D5.2** 고른 팀 기여 매트릭스: part-1 carving(hwanyeeeee) / part-3 EVTX·AI·UI(팀원) + 통합 지점
- **D5.3** (선택) 교차상관 분석 강화: Timeline 활동 ↔ EVTX 동시각 이벤트 연결 = 무료 단일도구 미존재 기능

## 범위 분담

- **우리(windows-timeline) 직접**: D2.1 버그, D3.1 해시(우리 출력측), D3.2 재현성, D3.3 carving 대조, D3.4 carving 오탐, D1 의의 문서, D5.1 carving 부각
- **팀원(wintrace) 조율/제안**: EVTX 해시, EVTX 대조(EvtxECmd/Event Viewer), 브루트포스 오탐, AI/UI 데모, exe 빌드
- **공동**: D4 발표 일체, D5.2 기여 매트릭스

## 검증 (각 항목이 5점 됐다는 증거)
- #1: 문제·의의 문서 + 시나리오에 named artifact·수사가치 명시 → "구체적·중요성 명확" 충족
- #2: OSError 0건(회귀 통과) + 실데이터 복원수치표 + 손상입력 graceful 로그
- #3: custody.json(해시 전후 일치) + 재현성 테스트 통과 + WxTCmd/FQLite 대조표(숫자) + precision/recall 표 + 한계 문서
- #4: 발표덱 + 동작 exe/영상 + Q&A 시트 + 리허설 시간 기록
- #5: 차별점 표(무료대비) + 학술 인용 + 기여 매트릭스
