# 우측 패널 UI 요구사항 (AI 조사관)

> **대상:** 대시보드 UI 담당
> **범위:** 화면 **우측 패널**만. 두 영역 = (A) 이상탐지 알림 + (B) 자연어 질의창.
> **연동:** 모든 데이터는 `window.pywebview.api.<메서드>()` 호출로만 받는다(아래 §3 계약). 내부 로직 몰라도 됨.

---

## 0. 한눈에

```
┌─────────────────────────────┐
│  [A] 이상탐지 알림            │  ← run_anomaly() 결과
│   • 탐지 N건 / 이상 없음       │
│   • high_risk 면 빨강 강조     │
├─────────────────────────────┤
│  [B] AI 조사관 질의창          │  ← ask(question) 결과
│   ┌───── 대화 영역 ─────┐     │
│   │ Q: 새벽에 로그인 실패? │     │
│   │ A: ...(answer)...     │     │
│   │  ▸ 근거 펼치기         │     │
│   └─────────────────────┘     │
│   [ 질문 입력...        ] [전송] │
└─────────────────────────────┘
```

모든 API 호출은 **비동기**(Promise) → `await` 또는 `.then()`. 호출 전 §4 "데이터 로드 선행" 확인.

---

## 1. [A] 이상탐지 알림 패널 (상단)

- **트리거:** 패널 마운트 시 자동 1회 호출, + 수동 "새로고침" 버튼(옵션).
  ```js
  const res = await window.pywebview.api.run_anomaly();
  ```
- **표시:**
  - `res.high_risk === true` → 패널 헤더 **빨강** + 경고 아이콘 (예: 감사로그 삭제 정황).
  - `res.anomalies.length > 0` → 탐지 건수 배지 + 각 항목 카드.
  - `res.anomalies.length === 0` → **녹색 "이상 징후 없음"**.
- **항목 카드 필드:** `time`(KST 문자열), `score`(0~1, 소수 2자리), `reason`(한 줄 한국어 설명). 클릭 시 §B 질의창에 "이 구간 자세히" 식으로 연결(옵션).
- **상태:** 로딩(스피너) / 정상 / 에러(`res.ok === false` 또는 throw → "분석 실패" 토스트).

## 2. [B] 자연어 질의창 (하단)

- **입력:** 텍스트 입력 + 전송 버튼. Enter=전송, Shift+Enter=줄바꿈.
- **호출:**
  ```js
  const res = await window.pywebview.api.ask(questionText);
  ```
- **⚠ 응답 지연:** 로컬 LLM(CPU 추론)이라 **수 초~수십 초** 걸림 → **로딩 스피너/“조사 중…” 필수**, 그동안 입력 비활성화.
- **표시(채팅식):**
  - 사용자 질문 버블 + AI 답변 버블(`res.answer`, 마크다운/줄바꿈 렌더 권장).
  - **접이식 "근거" 섹션**(기본 접힘):
    - `res.tool_calls` → 호출한 도구 목록(어떤 도구를 왜 썼는지, 감사 추적).
    - `res.evidence` → 근거 이벤트 목록(시각·EventID·계정 등 표/리스트).
- **상태:** 대기 / 응답중 / 에러("응답 실패, 재시도").
- **예시 질문(placeholder/칩 버튼):**
  - "로그인 실패가 많았던 계정 있어?"
  - "감사 로그가 삭제된 적 있어?"
  - "새벽 시간대 의심스러운 활동 보여줘"

## 3. JS API 계약 (응답 스키마 — 이대로 mock 가능)

> 엔진이 이 스키마를 보장한다. UI는 이 모양만 알면 됨.

### `run_anomaly()` → 이상탐지 결과
```json
{
  "ok": true,
  "high_risk": false,
  "anomalies": [
    { "window_id": "2020-09-09T03:00", "time": "2020-09-09 03:00 KST",
      "score": 0.87, "reason": "1시간 내 로그온 실패 23회 (평소 대비 급증)" }
  ]
}
```
- 이상 없음: `{ "ok": true, "high_risk": false, "anomalies": [] }`
- 실패: `{ "ok": false, "error": "사유" }`

### `ask(question: string)` → AI 조사관 답변
```json
{
  "answer": "새벽 2~3시에 IEUser 계정으로 로그온 실패가 23회 발생했습니다. ...",
  "tool_calls": [
    { "name": "analyze_logons", "arguments": { "start": "...", "end": "..." } }
  ],
  "evidence": [
    { "pk": 1, "time_kst": "2020-09-09T22:18:23+09:00",
      "event_id": 4625, "category": "logon_fail", "account": "IEUser" }
  ]
}
```
- 에러 시: `{ "answer": "조사 중 오류가 발생했습니다.", "tool_calls": [], "evidence": [] }`

> ⚠ 위 값은 **예시**. `evidence` 배열은 길 수 있음(스크롤/펼치기 처리). `tool_calls`는 0개일 수도 있음.

## 4. 데이터 로드 선행 (팀 확정 필요) ❗

`ask` / `run_anomaly` 는 **EVTX가 먼저 로드돼 DB에 적재된 상태**여야 동작. 로드 메서드:
```js
const res = await window.pywebview.api.load_evtx(path);  // { ok, records, stats:{ total, by_event_id } }
```
**질문: 이 "EVTX 로드"를 누가/어디서 부르나?**
- (가) 메인 앱/좌측 패널(1번 타임라인 담당)이 파일 열 때 호출 → 우측 패널은 질의만. **← 권장**
- (나) 우측 패널에도 "로그 불러오기" 버튼 필요.

우측 패널만 맡으면 보통 (가). 미로딩 상태면 패널에 **"먼저 EVTX 로그를 불러오세요"** 안내 띄우기.

## 5. 우측 패널 정리 (체크리스트)

- [ ] 우측 고정 패널(폭 예: 380~480px), 상단 [A]/하단 [B] 세로 배치
- [ ] [A] `run_anomaly()` 호출 + high_risk 강조 + 0건/N건/에러 상태
- [ ] [B] 입력창 + `ask()` 호출 + **로딩 스피너(긴 응답)** + 답변/근거(접이식)
- [ ] 모든 API `await`, 에러 try/catch
- [ ] 미로드 상태 안내 (§4)
- [ ] 색/폰트/레이아웃은 자유(엔진 무관). 위 필드만 맞춰주면 됨.

> 응답 스키마 변경/추가 필요하면 엔진 담당(나)에게 말해줘. `app.py` Api 클래스에서 맞춤.
