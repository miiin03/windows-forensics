# 우측 패널 UI 디자인 요구사항 (AI 조사관)

> **범위:** 화면 **우측 패널**만. 위=이상탐지 알림, 아래=AI 질의창.
> 데이터 연동/스키마는 별도(`REQUIREMENTS.md`). 이 문서는 **화면·UX 디자인만**.

---

## 1. 레이아웃

```
┌─────────────────────────────┐  우측 고정 패널
│ ⚠ 이상탐지            [새로고침]│  ← [A] 상단 (높이 자동, 접이식 가능)
│ ┌─────────────────────────┐ │
│ │ 카드: 시각 · 점수 · 사유   │ │
│ │ 카드: ...                │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ 🔍 AI 조사관                 │  ← [B] 하단 (남는 높이 전부)
│ ┌──── 대화 스크롤 영역 ────┐  │
│ │ Q: 새벽에 로그인 실패 있어?│  │
│ │ A: …답변…                │  │
│ │   ▸ 근거 보기            │  │
│ └─────────────────────────┘  │
│ [ 질문 입력…           ] [전송]│
└─────────────────────────────┘
```

- 우측 고정 폭 **380~480px** (좌측은 다른 팀원 영역, 안 건드림).
- 세로 2분할: **[A] 상단(콘텐츠만큼)**, **[B] 하단(나머지 높이 전부 차지 + 내부 스크롤)**.
- [A]/[B] 사이 구분선.

---

## 2. [A] 이상탐지 알림 (상단)

- **헤더:** 제목 "이상탐지" + 우측 작은 새로고침 버튼.
- **상태별 표시 (가장 중요):**

| 상태 | 디자인 |
|---|---|
| 🟢 이상 없음 | 녹색 톤, "이상 징후 없음" 한 줄 |
| 🟡 탐지됨 | 탐지 건수 배지 + 카드 리스트 |
| 🔴 고위험 | **헤더/테두리 빨강 + 경고 아이콘**, 가장 강조 |
| ⏳ 분석중 | 스피너 + "분석 중…" |
| ⚠ 에러 | "분석 실패" + 재시도 |

- **항목 카드:** `시각(KST)` · `점수(0~1, 색 게이지 권장)` · `사유 한 줄`. 점수 높을수록 진한 색.
- 카드 클릭 → (옵션) 아래 질의창에 관련 질문 자동 입력.

## 3. [B] AI 조사관 질의창 (하단)

- **대화 영역:** 채팅 버블. 사용자 질문(오른쪽 정렬) / AI 답변(왼쪽 정렬).
- **답변 버블 안 "근거 보기" 접이식**(기본 접힘): 펼치면 사용한 도구 + 근거 이벤트 목록(표/리스트). → "AI가 추측 아니라 실제 로그 근거로 답한다"는 신뢰감 디자인.
- **입력:** 하단 고정 입력창 + 전송 버튼. Enter=전송, Shift+Enter=줄바꿈.
- **⏳ 응답 지연 처리(필수):** 답변이 **수 초~수십 초** 걸림(로컬 AI). → 전송 즉시 **"조사 중…" 로딩 버블 + 입력 비활성화**. 빈 화면 금지.
- **예시 질문 칩**(입력창 위, 클릭 시 입력):
  - "로그인 실패 많은 계정?"
  - "감사 로그 삭제된 적 있어?"
  - "새벽 시간대 의심 활동 보여줘"
- **빈 상태:** 첫 진입 시 안내문 + 예시 칩.

---

## 4. 공통 디자인 원칙

- **색/폰트/테마는 자유** — 좌측 대시보드와 톤 맞추면 됨.
- **위험 신호 색 일관:** 정상=녹색, 주의=노랑/주황, 고위험=빨강.
- **로딩 상태 항상 표시** (이상탐지·질의 둘 다). 응답 느려도 멈춘 것처럼 보이면 안 됨.
- **포렌식 톤:** 신뢰감(근거 제시) · 간결 · 과한 애니메이션 지양.
- 반응형/접이식은 옵션. 우선순위는 위 상태 표시 + 로딩 처리.
